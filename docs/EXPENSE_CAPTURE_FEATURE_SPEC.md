# 経費入力・証憑AI入力 機能仕様

> 📌 **発展方針の正本は [`EVOLUTION_PLAN.md`](EVOLUTION_PLAN.md) です。** 本書はこの機能の有効な仕様詳細です（正本の「経費キャプチャ（AI証憑入力）」節と対応）。新FastAPIで実装します。

## 目的

`pseudo_freee` の経費入力を、実務で使いやすい入力体験へ拡張する。支払先・勘定科目・税区分を候補から選べるようにし、将来的には請求書・レシート画像からAIが入力候補を作る。

## 対象範囲

今回の機能群は以下の順で実装する。

1. 支払先・勘定科目・税区分の候補マスタ化
2. 経費入力フォームの検索付き候補入力化
3. 請求書・レシート画像アップロード
4. スマホのカメラ撮影入力
5. AI/OCRによる入力候補作成
6. 取引詳細で添付画像、AI抽出結果、ユーザー修正後データを確認

## 入力仕様

### 支払先

- 候補から選択できる。
- 入力文字で候補を絞り込める。
- 候補にない支払先はマスタ設定で追加する。
- 支払先はマスタ設定から事前追加できる。

### 勘定科目

- 候補から選択できる。
- 入力文字で候補を絞り込める。
- 候補にない勘定科目は、原則としてマスタ設定で追加する。
- 経費入力欄では、既存マスタから検索・選択する。
- 勘定科目マスタには標準税区分を持たせる。
- 経費入力で勘定科目を選択した場合、標準税区分を税区分欄に反映する。

初期候補:

- 消耗品費
- 旅費交通費
- 通信費
- 荷造運賃
- 支払手数料
- 広告宣伝費
- 会議費
- 接待交際費
- 水道光熱費
- 地代家賃
- 新聞図書費
- 修繕費
- 雑費
- 仕入高

### 税区分

- 候補数が少ないため、通常のドロップダウンで選択する。

初期候補:

- 課税仕入 10%
- 課税仕入 8%
- 対象外
- 非課税
- 不課税

## AI/OCR入力仕様

AI入力は、会計取引を直接登録せず、経費入力フォームに入力候補を反映する。

想定フロー:

```text
請求書・レシート画像をアップロード
  ↓
AI/OCRで内容を解析
  ↓
発生日、支払先、金額、税区分、摘要、勘定科目候補を推定
  ↓
フォームへ仮入力
  ↓
ユーザーが確認・修正
  ↓
経費として登録
```

AIが推定する項目:

- 発生日
- 支払先
- 金額
- 税区分
- 摘要
- 勘定科目候補
- 読み取り信頼度

実務上の方針:

- AI推定結果は必ずユーザー確認を挟む。
- 信頼度が低い項目はフォーム上で分かるようにする。
- 元画像、AI抽出結果、ユーザー修正後データを後から確認できるようにする。

## スマホ対応仕様

スマホでは、ファイル入力からカメラ撮影を起動できるようにする。

HTML方針:

```html
<input type="file" accept="image/*" capture="environment">
```

注意点:

- iPhone/Androidで挙動差がある。
- ブラウザによってはカメラ起動ではなく写真選択になる。
- ポートフォリオMVPでは、スマホで撮影または画像選択できる状態を合格ラインとする。

## データ保存方針

将来の証憑対応では、会計取引とは別に証憑テーブルを追加する。

想定項目:

- 取引ID
- ファイル名
- ファイル保存パス
- MIME type
- AI抽出JSON
- ユーザー修正後JSON
- 作成日時

## Git記録ポイント

機能の齟齬を防ぐため、以下の単位でGitに記録を残す。

1. 仕様書更新: この仕様書、要件定義書、プロダクト計画を更新した時点
2. 入力候補MVP: 支払先・勘定科目・税区分の候補入力が動いた時点
3. 証憑アップロード: 画像アップロードとスマホカメラ起動が動いた時点
4. AI入力候補: AI/OCRの解析結果をフォームに反映できた時点
5. 詳細確認: 添付画像、AI抽出結果、ユーザー修正後データを取引詳細で確認できた時点
