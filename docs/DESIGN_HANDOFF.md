# デザインAI向け受け渡し資料

## 目的

このアプリは、中小企業・小規模EC向けの在庫管理ダッシュボードです。

単なる一覧表示ではなく、以下を一画面で判断できる業務アプリとしてブラッシュアップしたいです。

- 現在庫と在庫金額
- 今月の仕入・売上
- 適正在庫シミュレーション
- 商品別の在庫元帳
- freee連携前の送信待ちキュー
- 送信前レビュー

## デザイン改善のゴール

「実務で毎日見る業務ダッシュボード」として、落ち着いていて、情報の優先順位が分かりやすく、入力・確認・判断がしやすい画面にしてください。

派手なランディングページ風ではなく、会計・在庫・業務管理ツールらしい、信頼感のあるUIを希望します。

## 触ってよい範囲

主に [app.py](app.py) 内の `INDEX_HTML` に含まれるHTML、CSS、JavaScriptの見た目・レイアウトを変更してください。

変更してよいもの:

- CSS
- セクション配置
- テーブルの見た目
- カードの見た目
- フォーム配置
- ボタン文言
- 見出し
- 補足説明文
- レスポンシブ対応

原則変更しないもの:

- Python側のDB処理
- APIエンドポイント
- SQLiteのテーブル構造
- 在庫計算ロジック
- freee送信用JSONの生成ロジック
- 予測・シミュレーション計算ロジック

## 現在の主要画面要素

以下の機能は画面から消さないでください。

| 要素 | 役割 |
|---|---|
| メトリクスカード | 在庫総額、商品数、発注/欠品リスク、今月仕入、今月売上を表示 |
| 在庫一覧 | 商品ごとの必要水準、現在在庫、状態、在庫金額、推奨発注量を表示 |
| 今月仕入 商品別 | 今月の商品別仕入数量・金額を表示 |
| 今月売上 商品別 | 今月の商品別売上数量・金額を表示 |
| 登録フォーム | 仕入明細・売上明細を登録 |
| 適正在庫シミュレーション | 予測期間を切り替えて、必要在庫・推奨発注量・月末不足を表示 |
| 在庫元帳 | 商品別の仕入・売上・取消・残高推移を表示 |
| freee送信待ちキュー | 会計連携前の送信待ちデータを表示 |
| 送信前レビュー | freee送信用の中間JSONを表示 |
| 最近の在庫移動 | 直近の在庫増減履歴を表示 |

## 特に改善してほしい点

- 情報量が多いので、セクションの優先順位を整理する
- 在庫一覧、適正在庫シミュレーション、在庫元帳の関係性を分かりやすくする
- 仕入・売上登録フォームが縦に長いため、入力しやすい配置にする
- テーブルが横に広くなるため、読みやすい余白・固定幅・横スクロールを検討する
- ステータスバッジを見やすくする
- 操作ボタンの意味を分かりやすくする
- 業務アプリらしい落ち着いた色にする
- モバイルよりも、まずはPCブラウザでの業務利用を優先する

## デザイン方針

望ましい方向:

- SaaS業務ツール風
- 会計・在庫管理らしい信頼感
- 情報密度は高めだが、詰まりすぎない
- テーブル中心
- 余白と見出しで視線誘導する
- 色は控えめに使い、重要状態だけ目立たせる

避けたい方向:

- ランディングページ風
- 派手なグラデーション
- 大きすぎるヒーロー
- 装飾過多
- カードだらけで業務情報が追いにくい構成

## 技術的な前提

現在はフレームワークなしのPython標準ライブラリだけで動いています。

- サーバー: `app.py`
- DB: SQLite `inventory.db`
- UI: `app.py` 内の `INDEX_HTML`
- 起動: `python app.py`
- URL: `http://127.0.0.1:8000`

外部パッケージを増やさずに、HTML/CSS/JavaScriptだけで改善できると最も安全です。
